## 1. Jupyter 노트북 실행 방식 (2026-04-05 변경)

> **토큰 효율 개선**: nbconvert 방식 → 하이브리드 방식으로 전환
> - 구 방식: Claude가 nbconvert로 직접 실행 → 모든 학습 로그가 컨텍스트 유입 (토큰 낭비, OOM/타임아웃 위험)
> - 신규 방식: USER 로컬 실행 + Claude Chrome 디버깅 → **약 70~80% 토큰 절감**

### 작업별 실행 주체

| 작업 유형 | 실행 주체 | 방법 |
|---|---|---|
| **무거운 훈련** (5-fold × 다중 모델, >10분) | **USER 로컬 실행** | Jupyter에서 직접 Run All |
| **Claude 디버깅/결과 확인** | Claude (Chrome MCP) | localhost:8888 접속, 셀 출력 확인 |
| **경량 실험** (<5분, 단순 검증) | Claude (Python 스크립트) | `python3 src/스크립트.py` |
| **피처 엔지니어링 코드 작성** | Claude (Write/Edit 도구) | `.py` 파일 직접 작성 |
| **제출 파일 생성** | Claude (Python 스크립트) | 모델 로드 후 예측, CSV 저장 |

### Chrome 디버깅 플로우
```
1. USER: 로컬 Jupyter에서 노트북 실행 (localhost:8888)
2. USER: "실행 완료, 확인해줘" 메시지
3. Claude: Chrome MCP로 localhost:8888 접속
4. Claude: 해당 노트북 열기 → 오류 셀 확인 → 결과 셀 파싱
5. Claude: 개선 코드 작성 후 보고
```

**Claude가 직접 실행 가능한 경우** (기존 nbconvert 방식 유지):
- ✅ 단일 Python 스크립트 (결과 출력 간결함)
- ✅ 검증/체크 코드 (assert, 통계 확인 등)
- ✅ 제출 파일 생성
- ❌ 5-fold 이상 대규모 모델 학습 (OOM/타임아웃 위험)

## 2. 자율 추진 지침 (토큰 여유 시 계획 외 업무 자동 진행)

**기본 원칙**: 사용자가 "계속해줘" 또는 다른 지시를 내리기 전에도, 토큰 여유가 있다면 **다음 우선순위에 따라 자율적으로 업무를 추진**합니다.

### 자율 추진 우선순위

```
1순위: 당일 스케줄 미완료 항목
2순위: 노트북 실행 및 결과 저장
3순위: 다음 날 작업을 미리 준비 (코드 초안 작성)
4순위: Notion 동기화
5순위: README Approach Log 업데이트
```

### 자율 추진 가능 업무 목록

| 업무 | 설명 | 자율 여부 |
|---|---|---|
| 노트북 셀 코드 작성 | EDA/Baseline/FE 코드 작성 | ✅ 자율 |
| 노트북 실행 (`nbconvert`) | 전체 노트북 실행 + 오류 체크 | ✅ 자율 |
| 피처 엔지니어링 코드 작성 | `src/feature_engineering.py` 작성 | ✅ 자율 |
| 실험 결과 기록 | README Approach Log 업데이트 | ✅ 자율 |
| 다음 날 노트북 초안 준비 | 다음 일정 코드 미리 작성 | ✅ 자율 |
| Notion 페이지 업데이트 | 진행 상황 동기화 | ✅ 자율 |
| 제출 파일 생성 | `submissions/` 에 CSV 저장 | ✅ 자율 |
| Public LB 제출 | 데이콘 사이트 직접 접속 | ❌ 사용자 직접 |
| 하이퍼파라미터 대규모 탐색 | 1시간+ 소요 작업 | ⚠️ 사전 확인 필요 |

### 자율 추진 시 보고 방식

업무 완료 후 반드시 아래 형식으로 보고:
```
✅ 자율 추진 완료: [업무명]
   - 결과: [핵심 수치/변경 내용]
   - 다음 할 일: [이어서 할 작업]
```
